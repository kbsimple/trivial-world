| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Chip gap in `packChipRow`, chip vertical padding |

| lg | 24px | Footer bottom padding, section bottom margin |
| xl | 32px | — |

Exceptions:

## Component Inventory

### setup.tsx — Player Row (modified)

**Row 1 (unchanged):** color dot (24px circle) + TextInput (player name, 18px) + remove button (×, 24px, `#ff6b6b`)

**Row 2 — Chip row (modified):**

Replace the existing two-chip row (pack chip + difficulty chip) with a single **Shared/Custom toggle chip**:

```
packChipRow (paddingLeft: 36, marginTop: 4, gap: 4, flexDirection: row)
  └── [Shared|Custom] chip  (packChip + conditional packChipDefault|packChipActive)
```

The difficulty chip is **removed entirely** from the player row. (Decision A1 — auto mode: remove difficulty chip to eliminate all inline Alert/Modal pickers. Difficulty-per-player via dedicated screen is deferred to a future phase.)

#### Toggle Chip States

| Player state | Chip label | Chip style | Tap action |
|---|---|---|---|
| `packId == null && comboId == null` | `"Shared"` | `packChipDefault` (`rgba(255,255,255,0.12)`) | Navigate: `router.push({ pathname: '/packs', params: { targetPlayerId: player.id } })` |
| `packId != null` (pack selected) | `"Custom: [name ≤12 chars]"` | `packChipActive` (`rgba(255,255,255,0.28)`) | Clear immediately: `updatePlayerPack(playerId, null)` + `updatePlayerCombo(playerId, null)` |
| `comboId != null` (combo selected) | `"Custom: [name ≤12 chars]"` | `packChipActive` (`rgba(255,255,255,0.28)`) | Clear immediately: `updatePlayerPack(playerId, null)` + `updatePlayerCombo(playerId, null)` |

Name truncation: `displayName.length > 12 ? displayName.slice(0, 12) + '…' : displayName`

No confirmation dialog for revert to Shared — immediate, no modal. (Source: locked decision from 21-RESEARCH.md and prompt key decisions #4.)

Remove handlers: `handlePickSource`, `handlePickDifficulty`, `webPicker` state, web picker Modal, and all related styles (`webPickerBackdrop`, `webPickerCard`, `webPickerTitle`, `webPickerDivider`, `webPickerItem`, `webPickerItemPressed`, `webPickerItemText`).

### packs/index.tsx — Per-Player Mode

When `targetPlayerId` is present (URL param from setup → /packs navigation):

**Title:** `"Select Pack for [player.name]"` — look up player name from `usePlayerStore`. When `targetPlayerId` is absent, title remains `"Select Question Pack"` (unchanged).

**Pack selection behavior:**
- Tapping a downloaded pack calls `updatePlayerPack(targetPlayerId, packId)` then `router.back()`
- `PackDetailsModal` onSelect calls same per-player path
- `router.push` is used from setup (not replace), so `router.back()` reliably returns to setup

**Combo selection (Decision A2 — auto mode):**
Show saved combos as a selectable section below the pack list when `targetPlayerId` is set. Each saved combo renders as a tappable row; tapping calls `updatePlayerCombo(targetPlayerId, comboId)` then `router.back()`. No navigation to `/packs/combos` needed for combo selection in per-player mode.

Combo section header: `"Saved Combos"` — shown only when `savedCombos.length > 0` and `targetPlayerId` is set.

**Multi-pack footer (Decision A3 — auto mode):**
Hide the multi-pack selection footer (`"Play with N packs →"` button and `"Clear selection"` button) when `targetPlayerId` is set. Single-pack selection only per player. The `selectedPackIds` multi-select state is not used in per-player mode.

When `targetPlayerId` is set, replace the default footer (Manage Combos + Home) with a **Back button only**:
```
"Back"  →  router.back()
```

When `targetPlayerId` is absent, all existing footer behavior is unchanged.

## Interaction Contracts

### Navigation Flow

```
setup.tsx player row
  └── "Shared" chip tap → router.push('/packs', { targetPlayerId })
        └── /packs/index.tsx (per-player mode)
              ├── tap pack → updatePlayerPack(targetPlayerId, id) → router.back()
              ├── tap combo → updatePlayerCombo(targetPlayerId, id) → router.back()
              └── "Back" footer → router.back()
  └── "Custom: [name]" chip tap → updatePlayerPack(id, null) + updatePlayerCombo(id, null)
        (immediate, no confirmation, no navigation)
```

### State Reactivity

`setup.tsx` reads `player.packId` and `player.comboId` from `usePlayerStore`. When `/packs` writes to `playerStore` and calls `router.back()`, `setup.tsx` re-renders with updated player state automatically — no polling, no event bus.

### allPlayersCustom Logic

No changes. Existing computation in `setup.tsx` remains:
```typescript
const allPlayersCustom = players.length > 0 && players.every(p => p.packId !== null || p.comboId !== null);
```
The Start Game button enable/disable logic and CONF-01 bypass are unchanged.
