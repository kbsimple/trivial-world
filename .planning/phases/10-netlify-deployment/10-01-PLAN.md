<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Repository → Netlify Build | Code pushed to main branch triggers untrusted build |
| Netlify Build → Netlify CDN | Build artifacts deployed to public CDN |
| Client Browser → Netlify CDN | Public HTTPS access to static sites |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | Build pipeline | mitigate | Use pnpm lockfile for dependency integrity, Turborepo handles build reproducibility |
| T-10-02 | Tampering | Static content | accept | No dynamic content, static sites cannot be modified at runtime |
| T-10-03 | Denial | Netlify CDN | accept | Netlify provides DDoS protection, out of scope for app configuration |
| T-10-04 | Tampering | SPA redirects | mitigate | Security headers in netlify.toml (X-Frame-Options, X-Content-Type-Options) prevent clickjacking and MIME sniffing |
| T-10-05 | Information Disclosure | Build logs | accept | Build logs may contain dependency versions, acceptable for public repo |
</threat_model>
