# Phase 7 — UI Design Contract

> Visual and interaction contract for the Question Generator Web App. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | **Decision required: shadcn/ui or Tamagui** |
| Preset | Pending design system decision |
| Component library | Radix UI (if shadcn) / Tamagui primitives (if Tamagui) |
| Icon library | lucide-react (if shadcn) / @tamagui/lucide (if Tamagui) |
| Font | Inter (consistent with mobile app) |

**Rationale for decision:**
- **shadcn/ui**: Better fit for Next.js web app, lighter bundle for static export, Tailwind-based, component copy-paste model
- **Tamagui**: Consistent with mobile app, shared token system, but designed for React Native primarily

**Recommendation from RESEARCH.md:** Tamagui 2.x for consistency with mobile app styling.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, form groups |
| xl | 32px | Layout gaps, page sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- CategoryBadge paddingHorizontal: 16px (matches md token)
- CategoryBadge paddingVertical: 8px (matches sm token)
- Reveal button minWidth: 200px (touch target accessibility)

## Interaction Patterns

### Question Generation Flow

1. User enters topic (required)
2. User selects category (required)
3. User optionally adds guidance
4. User clicks "Generate Questions"
5. Progress indicator shows: "Generating question 1/N..."
6. Each question goes through 3-pass verification
7. Progress shows: "Question 1/N — Verifying (2/3)"
8. On completion, user is navigated to Review page

### Review Flow

1. Questions presented one at a time (single-question focus per D-11)
2. Confidence badge shows verification score (D-10)
3. All 3 verification pass results visible (D-09)
4. User can:
   - **Approve**: Adds question to approved list
   - **Edit**: Modify question/answer/difficulty, then approve
   - **Reject**: Discard question (with confirmation)
5. Navigation moves through queue with Prev/Next

### Pack Export Flow

1. User reviews pack metadata (name, description, author)
2. Category distribution shown automatically
3. User clicks "Download Pack"
4. JSON file generated with schema validation
5. Browser downloads file
