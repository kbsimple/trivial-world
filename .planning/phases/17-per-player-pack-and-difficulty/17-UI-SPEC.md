## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Gap between pack chip and difficulty chip on the same row |
| sm | 8px | Chip horizontal padding (`paddingHorizontal: 8`) |
| md | 16px | Container padding, horizontal padding on player rows |
| lg | 24px | Section margin between player list and Add Player button |
| xl | 32px | — |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions:
- Pack chip row left padding: 36px (existing `packChipRow.paddingLeft`; preserves alignment with name input, accounting for 24px color dot + 12px gap). The difficulty chip inherits this row — no change to the indent.
- Pack chip and difficulty chip vertical padding: 4px (existing `packChip.paddingVertical`). This is intentionally sub-scale for dense in-row chip sizing.
- Chip border radius: 6px (existing `packChip.borderRadius`). Below the 8px default to visually distinguish from the card-level 8px radius.
- Progress strip gap between entries: 4px (existing `progressStrip.gap`). Dense list — sub-scale is intentional.
- Color indicator in player row: 24px diameter (existing `colorIndicator`). Fixed size, not a spacing token.

Source: `apps/mobile/app/game/setup.tsx` and `apps/mobile/app/game/turn.tsx` (verified).

## Component Inventory

### New: DifficultyChip (inline — no separate file)

Implemented as an inline `Pressable` + `Text` inside `setup.tsx`, following the identical pattern as the existing pack chip. No new component file needed.

```
Pressable
  style: styles.packChip + (difficultyPreference ? styles.packChipActive : styles.packChipDefault)
  onPress: () => handlePickDifficulty(player.id)
  disabled: Platform.OS === 'web'  ← guard: chip never renders on web
Text
  style: styles.packChipText  ← reuse existing style, no new style needed
  numberOfLines: 1
  content: player.difficultyPreference
    ? capitalize(player.difficultyPreference)   → 'Easy' | 'Medium' | 'Hard'
    : 'Any Difficulty'
```

States:
- Default (null/unset): chip uses `packChipDefault` style; label is "Any Difficulty"
- Active (easy/medium/hard): chip uses `packChipActive` style; label is "Easy", "Medium", or "Hard"
- Pressed: `opacity: 0.7` (inherit from Pressable `pressed` state — existing pattern in turn.tsx)
- Web: chip is not rendered (`Platform.OS !== 'web'` guard — same as pack chip)

### Modified: packChipRow (setup.tsx)

The existing `packChipRow` view becomes a flex row containing both the pack chip and the difficulty chip side by side:

```
View  (packChipRow: flexDirection row, paddingLeft 36, marginTop 4, gap 4)
  Pressable  (pack chip — existing, unchanged)
  Pressable  (difficulty chip — NEW)
```

No layout change to `packChipRow` style needed other than confirming `flexDirection: 'row'` (already set) and adding `gap: 4` between chips. Both chips use the same `packChip`, `packChipDefault`, `packChipActive`, and `packChipText` styles — no new styles.

### Modified: progressEntry (turn.tsx)

Add a difficulty label text node between `displayPackName` and `progressCount`:

```
View  (progressEntry: flexDirection row, gap 8)
  View   (progressDot)
  Text   (progressName — player name)
  Text?  (progressPack — pack name, conditional)
  Text?  (progressDifficulty — difficulty label, conditional)  ← NEW
  Text   (progressCount)
```

New style `progressDifficulty`: `{ color: '#888', fontSize: 11, marginRight: 4 }` — identical to `progressPack`. Render only when `difficultyLabel != null`.

Difficulty label values: `'Easy'` | `'Medium'` | `'Hard'` (capitalized). Never display `'Any Difficulty'` in the progress strip — null difficulty means no label shown (clean, uncluttered strip).

## Interaction Contracts

### Difficulty Picker (handlePickDifficulty)

Triggered by tapping the difficulty chip. Uses `Alert.alert` — same mechanism as `handlePickPack`.

```
Alert.alert(
  title:   'Select Difficulty'
  message: undefined
  buttons: [
    { text: 'Any Difficulty', onPress: () => updatePlayerDifficulty(playerId, null) }
    { text: 'Easy',           onPress: () => updatePlayerDifficulty(playerId, 'easy') }
    { text: 'Medium',         onPress: () => updatePlayerDifficulty(playerId, 'medium') }
    { text: 'Hard',           onPress: () => updatePlayerDifficulty(playerId, 'hard') }
    { text: 'Cancel',         style: 'cancel' }
  ]
)
```

Platform guard: `if (Platform.OS === 'web') return;` at the top of `handlePickDifficulty` (same as `handlePickPack`).

Selection feedback: `Alert.alert` closes automatically on option tap. The chip label and background update synchronously through Zustand state (existing pattern — no loading state needed).

### No new navigation. No new screens. No new modals.

## Visual Layout (ASCII)

### Setup Screen — Player Row (native, after Phase 17)

```
┌──────────────────────────────────────────────────────────────┐
│ ● [Player Name Input                           ] [ × ]       │
│   [Default        ] [Any Difficulty]                         │
│   ↑ pack chip       ↑ difficulty chip (NEW)                  │
└──────────────────────────────────────────────────────────────┘
```

- Both chips share row 2 (`packChipRow`), separated by 4px gap.
- Pack chip is left; difficulty chip is right.
- Both chips truncate to `numberOfLines={1}` if text overflows.
- Row 2 is indented 36px from left edge (aligns with name input start, past the color dot + gap).
- On web: row 2 is not rendered at all (both pack and difficulty chips are native-only).

### Turn Screen — Progress Strip Entry (native, after Phase 17)

```
● PlayerName        Default    Easy    2/6
  ↑ dot  ↑ name     ↑ pack     ↑ diff  ↑ count
```

- Difficulty label appears only when `playerDifficulties[idx] != null`.
- Pack name appears only when player has a custom pack (unchanged behavior).
- Entry is a single flex row with `gap: 8`; all text is 11px.
