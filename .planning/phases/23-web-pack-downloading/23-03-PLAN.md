<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| IDB read → application | Data read from IDB is previously-validated Question[] or PackIndexEntry[] — same trust level as in-memory cache |
| network → IDB | Only Zod-validated data is written to IDB (safeParse must succeed) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-03-01 | Tampering | IDB read in fetchWebPackQuestions | accept | Data was Zod-validated on write. Same-origin IDB cannot be tampered from outside the app. |
| T-23-03-02 | Spoofing | fetchPackIndex offline fallback | accept | IDB fallback data was fetched from the same GENERATOR_PACK_INDEX_URL and Zod-validated on write. Offline attacker cannot modify same-origin IDB. |
| T-23-03-03 | Information Disclosure | console.warn on offline | accept | Logs "offline — serving cached" to console — no sensitive data exposed. |
</threat_model>
