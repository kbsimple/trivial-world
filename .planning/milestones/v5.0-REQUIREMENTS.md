## v5.0 Requirements (ROADMAP Success Criteria)

### Phase 16: CLI Bulk Question Generation

| ID | Requirement | Status | Outcome |
|----|------------|--------|---------|
| SC-16-1 | CLI generates full pack (all 6 categories) from --topic | ✅ Satisfied | `generate.ts` with `ALL_CATEGORIES` iteration |
| SC-16-2 | Questions saved immediately as draft JSON (streaming) | ✅ Satisfied | `appendDraftQuestion` called inside per-question loop |
| SC-16-3 | Each question has tidbits field (non-empty string) | ✅ Satisfied | `buildCLIQuestionPrompt` with REQUIRED instruction + QuestionSchema coercion |

| SC-16-5 | Published packs in pack index, downloadable | ✅ Satisfied | `publishDraft()` writes to `public/packs/` + `packs.json` |
| SC-16-6 | tidbits field displayed in question reveal screen | ✅ Satisfied | `QuestionCard.tsx` renders tidbits when `revealed && tidbits` |

### Phase 17: Per-Player Pack and Difficulty

| ID | Requirement | Status | Outcome |
|----|------------|--------|---------|
| SC-17-1 | Each player selects pack AND difficulty independently in setup | ✅ Satisfied | Two chips per player row in `setup.tsx` (native) |
| SC-17-2 | Game draws from multiple packs simultaneously | ✅ Satisfied | `playerPackIds` snapshot at `startGame()`; `selectCategory` reads per-player packId |
| SC-17-3 | Questions served from each player's pack + difficulty combo | ✅ Satisfied | Full chain: `selectCategory → selectQuestion(category, packId, difficulty)` |
| SC-17-4 | Game-level defaults serve as fallback | ✅ Satisfied | `packId ?? activePackId`; `difficulty != null ? [d] : enabledDifficulties` |
| SC-17-5 | UI shows each player's pack and difficulty config | ✅ Satisfied | Setup chips + turn.tsx progress strip difficulty label |
