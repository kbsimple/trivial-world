## Tech Debt

None of these are blockers. All active code paths are correct. These items pose risk only if dead code is activated in a future refactor.

### Phase 16

**TD-16-1 (High Priority): `questionProvider.getNextQuestionFromDatabase` is dead code with two correctness bugs**

`questionProvider.getNextQuestionFromDatabase` is never called from the active mobile code path (which uses `questionStore.selectQuestion`'s inline WatermelonDB query). The function has two bugs:
1. Its return value omits `tidbits` — would break SC-16-6 if activated
2. Its signature has no `packId` parameter — would break SC-17-2 and SC-17-3 if activated

The two mobile implementations (`questionStore` inline vs `questionProvider.getNextQuestionFromDatabase`) must be reconciled. Options:
- Route `questionStore` native path through `questionProvider` (removes duplication, requires fixing both bugs first)
- Fix `questionProvider.getNextQuestionFromDatabase` and add a comment marking it as the canonical implementation (safeguards future refactor)

**TD-16-2 (Low Priority): Starter pack has no tidbits content**

The default/starter pack `trivial-world-starter-7f3a9c2e.json` predates Phase 16. It has zero tidbits fields. Players using the default pack will not see the tidbits reveal UI. The code path is correct; this is a content gap only. Resolution: regenerate the starter pack with the Phase 16 CLI and replace the bundled JSON.

### Phase 17

**TD-17-1 (Low Priority): `effectiveDifficulties` derivation logic duplicated**

Identical 3-line logic exists in `questionStore.ts` (lines 123–127) and `questionProvider.ts` (lines 187–190). If the precedence rule changes, both copies must be updated. Extract to a shared `resolveEffectiveDifficulties(difficulty, enabledDifficulties)` helper in a utilities module.

**TD-17-2 (Low Priority): `capitalize` inline duplication with double-cast in `turn.tsx`**

`difficultyLabel` computation is written inline in `setup.tsx` and `turn.tsx`. `turn.tsx` casts `playerDifficulties[idx]` twice as `string` in the same expression. Extract a `capitalize` helper (or use a single const) to remove duplication and the double cast.
