# Milestone v5.0: Content Generation Tooling

**Status:** ✅ SHIPPED 2026-06-13
**Phases:** 16–17
**Total Plans:** 6

## Overview

v5.0 added a CLI pipeline for bulk AI question generation with per-question tidbits, and per-player difficulty configuration that extends the v4.0 per-player pack selection. Conductors can now generate full question packs from a topic via CLI, review and publish them, and players can independently configure both their pack and difficulty level in-game.

## Milestone Summary

### Key Decisions

- D-v5-01: Per-player difficulty preference — each player can independently set Easy/Medium/Hard or Any (null)
- D-v5-02: playerDifficulties snapshotted at startGame() alongside playerPackIds for immutable in-game state
- D-v5-03: Per-player difficulty overrides game-level enabledDifficulties; null falls back to game-level filter
- D-v5-04: CLI pipeline (generate.ts + review.ts) for bulk question generation with tidbits field
- D-v5-05: tidbits field in QuestionSchema (v3), WatermelonDB migration 003, displayed in answer reveal
- D-v5-06: Draft JSON workflow — generate writes immediately, review decouples inspection and publish
- D-v5-07: `effectiveDifficulties = difficulty != null ? [difficulty] : enabledDifficulties` priority pattern

### Issues Resolved

- Phase 16 initial verification: 11/13 must-haves. Gap closure added `buildCLIQuestionPrompt` to `prompts.ts` and wired `generate.ts` to use it directly with `generateObject` (not `generateAndVerifyQuestion`)
- Phase 17 UI-SPEC blocked by typography: collapsed 5 font sizes → 4 and 3 font weights → 2 on first checker run

- `effectiveDifficulties` logic duplicated in `questionStore.ts` and `questionProvider.ts` — extract to shared helper
- Pre-close open items from v3.0/v4.0: Phase 10 UAT (0 pending scenarios), Phase 10+15 human_needed verifications, 5 deployment/testing todos from 2026-06-10

### Technical Debt Incurred
