# Milestone v11.0 Audit: Web Pack Downloading

**Shipped:** 2026-06-18
**Phase:** 23
**Plans:** 6/6 complete

## Summary

Phase 23 delivered full web offline support for the Trivial World app:

- **Service Worker (Workbox):** `sw-template.js` + `build-sw.mjs` post-build script. Stale-while-revalidate strategy for `/api/v1/packs.json` index. Precaches app shell (JS/CSS/HTML/images). Silent update via SKIP_WAITING. Registered in `public/index.html` via workbox-window CDN v7.4.0.

- **IndexedDB (idb-keyval v6.2.5):** `packCache.ts` platform shim + `packCache.web.ts` adapter with 9 exported functions. Key scheme: `pack:{packId}` → Question[], `pack-checksum:{packId}` → string, `pack-index` → PackIndexEntry[]. Store: `trivial-world-packs / pack-cache`.

- **IDB-first question fetch:** `questionProvider.ts` checks IDB before network. `packIndex.ts` writes to IDB on success and falls back to IDB cache on network failure (offline). Old in-memory `webPackCache` Map removed.

- **Pack store actions:** `offlinePackIds`, `downloadPackForOffline(entry)` (streaming, checksum-skip, checksum validation), `refreshOfflinePackIds()`. `offlinePackIds` persisted in Zustand partialize.

- **Download UI:** `packs/index.tsx` shows "Download for offline" link per pack row on web; "Downloaded" italic badge when cached. `refreshOfflinePackIds()` called on mount. `DownloadProgress` shown during web offline downloads.

- **Tests:** 400 tests passing across 14 test files (+98 from v10.0 baseline of 302).
